# ADPI Manager (Менеджер АДПИ) 🧯

**ADPI Manager** — это десктопное приложение на Python, разработанное для учета и мониторинга установки автономных дымовых пожарных извещателей (АДПИ) в жилом секторе. Программа позволяет вести реестр семей, отслеживать сроки проверок оборудования и формировать отчетную документацию.

## ✨ Основные возможности

* **Управление реестром:** Создание, редактирование, архивация и удаление карточек жильцов.
* **Умные фильтры:**
    * Поиск по ФИО и адресу.
    * Фильтрация по статусу (Активные/Архив) и типу населенного пункта (Город/Деревня).
    * Отбор по количеству детей и количеству установленных датчиков.
    * Фильтр по датам установки.
* **Контроль проверок:** Автоматическое выделение «просроченных» записей (если проверка не проводилась более 365 дней).
* **Экспорт данных:**
    * **Excel:** Полная выгрузка базы данных для аналитики.
    * **Word:** Генерация готовых актов установки/списков обхода по установленному образцу.
* **История перемещений:** Автоматическое сохранение предыдущих адресов при смене места жительства резидента.
* **Локальное хранение:** Автоматическое сохранение данных в JSON-реестр и загрузка последнего измененного файла при старте.

## 🛠 Технологический стек

* **Интерфейс:** `customtkinter` (современный UI в стиле Dark/Light mode).
* **Работа с данными:** `pandas`, `json`.
* **Генерация документов:** `python-docx` (Word), `openpyxl` (Excel).
* **Сборка:** Поддержка путей для `PyInstaller` (создание единого .exe файла).

## 🚀 Быстрый старт

### Требования
Убедитесь, что у вас установлен Python 3.9+ и необходимые библиотеки:
```bash
pip install customtkinter pandas python-docx openpyxl
```

### Запуск
```bash
python ResidentCards.py
```
Либо скачайте .exe с этого репозитория.

## 📂 Структура данных
Приложение автоматически создает папку `registry` в корневой директории. В ней хранятся `.json` файлы с базой данных. 
Пример структуры объекта в базе:
```json
{
    "id": 1,
    "resident_name": "Иванов Иван Иванович",
    "children_count": 3,
    "location": {
        "raw_address": "ул. Ленина, д. 10",
        "is_village": false
    },
    "device": {
        "model": "Рубеж",
        "count": 2,
        "install_date": "01.01.2024"
    },
    "status": "active"
}
```

## 📝 Особенности интерфейса
* **Цветовая индикация:** Активные записи помечены зеленым маркером, архивные — красным.
* **Подсказки:** Наведите на значок `[?]` в карточке, чтобы быстро прочитать технические или социальные заметки без открытия окна редактирования.
* **Динамическая загрузка:** Список подгружается постепенно (по 25 записей) при скроллинге, что обеспечивает высокую скорость работы даже с большими базами данных.

## 📦 Сборка в EXE
Для создания исполняемого файла используйте команду:
```bash
pyinstaller --noconsole --onefile --add-data "registry;registry" ResidentCards.py
```

## 🛠 Вспомогательные инструменты (Utilities)

В состав проекта входят две специализированные утилиты для подготовки и поддержки базы данных.

### 1. Конвертер таблиц (`adpi_JsonConverter.py`)
Этот инструмент предназначен для первичного преобразования разрозненных Excel-отчетов в структурированный JSON-реестр.

* **Автоматическое сопоставление (v4.6 Auto-Header):** Система сама ищет строку заголовков в Excel-файле, распознавая колонки (ФИО, Адрес, Даты) по ключевым словам.
* **Интеллектуальный парсинг адреса:** Скрипт разбирает сырую строку адреса, отделяя город от деревни, выделяя улицу, номер дома и квартиру.
* **Анализ статуса:** Автоматически присваивает теги (например, `functional`, `maintenance_required`) и меняет статус семьи на основе текстовых заметок в таблице (например, если написано «переехали» или «отказ»).
* **Дедупликация:** При импорте данные автоматически объединяются. Если одна и та же семья встречается дважды, программа объединяет их историю проверок и сохраняет актуальный адрес.

### 2. Синхронизатор баз (`adpi_synchronizator.py`)
Инструмент для «умного» слияния двух JSON-файлов, когда нужно обновить текущую базу данными из нового отчета, не потеряв при этом накопленную историю.

* **Приоритетное слияние:** Обновляет информацию о жильцах, используя новый файл как приоритетный источник, но сохраняя уникальные данные из архива.
* **Глубокая синхронизация истории:**
    * **Даты проверок:** Объединяет списки дат из обоих файлов, исключая дубликаты и сортируя их по хронологии.
    * **История адресов:** Если адрес в новом файле изменился, старый адрес автоматически отправляется в массив `location_history` с пометкой о дате изменения.
* **Обогащение заметок:** Если в новом отчете социальные или технические заметки пусты, скрипт автоматически подтянет их из старой базы.
* **Контроль удалений:** Утилита наглядно показывает, какие записи присутствовали в старой базе, но отсутствуют в новой (помогает отслеживать снятие семей с учета).
